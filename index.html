What is Havoc Preset?

Havoc Preset is a Roblox key system UI library. Loaded via game:HttpGet() and configured entirely from your own script. Once a valid key is redeemed, getgenv().SCRIPT_KEY is set to the validated key string and your game script continues.

Tip — The library is CDN-hosted. You only write a loader script with your config — Jnkie.com handles the library internals. Never modify the library directly.
02 — Flow
How It Works
1
Load the Library
Your script calls loadstring(game:HttpGet(LIBRARY_URL))() — fetches the library from CDN and returns a Hub object.
2
Configure & Init
Pass your Config to Hub.new(Config) then call hub:Init() to render the key system UI.
3
Wait for Redemption
while not getgenv().SCRIPT_KEY do task.wait(0.1) end — blocks execution until a valid key is verified. SCRIPT_KEY is set to the validated key string (or "KEYLESS" for keyless scripts).
03 — Template
Minimal Template
Lua
getgenv().SCRIPT_KEY = nil

local Hub
pcall(function()
  local src = game:HttpGet("LIBRARY_URL")
  Hub = loadstring(src)()
end)
if type(Hub) ~= "table" then return end

local Config = { -- your config here }
local hub = Hub.new(Config)
hub:Init()

while not getgenv().SCRIPT_KEY do
  task.wait(0.1)
end
-- game script runs here
04 — Core
Junkie_Configs
Controls Junkie SDK integration — key validation, webhook relay, and provider setup.
PropertyTypeDescription
UseJunkiebooleantrue loads the Junkie SDK for real-time key validation. false accepts any key (testing mode).
Junkie_WebhookstringYour Junkie relay webhook URL. Notifies you on key redemption. Leave "" to disable.
ServicestringYour service name from the Junkie dashboard. Must match exactly.
IdentifierstringYour user/dashboard identifier from Junkie.
ProviderstringThe provider name configured in your Junkie dashboard.
MaxKeyAttemptsnumberFailed attempts before UI force-closes. Recommended: 5.
Premiumbooleantrue loads the Premium script URL from GameSupport instead of Free.
Providers
Providers
Array of key provider buttons shown in the UI. Each can be shown, hidden, or pinned.
PropertyTypeDescription
NamestringDisplay name on the provider button.
Iconstringrbxassetid for the icon. Use "rbxassetid://0" for the built-in default.
UrlstringURL opened on click. Fallback if get_key_link() returns nil.
Showbooleantrue = visible.
IsJunkiebooleanExactly ONE provider should have this. Enables SDK's get_key_link() flow.
Pinnedbooleantrue = rendered first with a highlight border.
Warning — Only one provider should have IsJunkie = true. If none, all providers render as regular URL buttons.
GameSupport
GameSupport
Maps Roblox Place IDs to script URLs. After key acceptance the matching script loads. Universal is required as a fallback.
Lua
GameSupport = {
  [2788229376] = {
    Name    = "Game Name Here",
    Free    = "https://api.jnkie.com/.../HASH/download",
    Premium = "https://api.jnkie.com/.../HASH/download",
  },
  Universal = {
    Name    = "Universal",
    Free    = "https://api.jnkie.com/.../HASH/download",
    Premium = "https://api.jnkie.com/.../HASH/download",
  },
}
Warning — Replace all HASH placeholders with your actual Junkie script hashes. The Universal entry is required as a fallback.
05 — Visual
UI_Configs
Controls the visual appearance and behaviour of the key system UI. Split into focused sub-sections below.
Navigate using the sidebar or Next button to reach General, Background, Snow, Key System, Discord, Socials, Instructions, and Changelog sub-sections.
General
General Settings
PropertyTypeDescription

  if not fn then return end

end)

local autoLoginResult = TryAutoLogin() -- handles saved key / SCRIPT_KEY env var

  pcall(function()
    local hub = Hub.new(Config)
    hub:Init()

  task.wait(0.1)

Note — The functions hasFileSystemSupport, saveVerifiedKey, loadVerifiedKey, and TryAutoLogin are already defined in the library template above Config. Do not remove them.
